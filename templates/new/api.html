Greenaware API Documentation

Welcome to the Greenaware API documentation! Our API empowers developers to integrate weather intelligence into their applications, providing access to real-time and forecast weather data for locations around the world. Below, you'll find comprehensive documentation on how to use our API to retrieve weather information and leverage powerful features to enhance your applications.

Please, Sign up to use our fast and easy-to-work weather APIs

Base URL

https://api.greenaware.com/v1

Authentication

Authorization: Bearer YOUR_API_KEY

Endpoints

  • GET /users

    Retrieve all the users with diverse roles .

  • POST /users

    Create a user with and specify the user's role (subscriber or observer).

  • GET /current-user

    Retrieve the current user data by specifying the token .

  • POST /auth

    Authenticate and Authorize users with the auth resource

  • POST /auth/activate-user

    Authenticate and Authorize users with the auth resource

  • GET /observations

    Retrieve current weather data for a specific location.

  • POST /observations

    Create weather observation data for a specific location.

  • PATCH /observations/{id}

    Ammend weather observation data for a specific location.

  • DELETE /observations/{id}

    An observer can delete his weather observation data for a specific location.

Query Parameters

  • Location:

    The location for which you want to retrieve weather data. This can be specified as a city name, ZIP code, or latitude/longitude coordinates.

  • units:

    The units in which you want to receive the weather data (e.g., metric or imperial).

  • start_date (for historical endpoint):

    The start date for the historical weather data you want to retrieve.

  • end_date (for historical endpoint):

    The end date for the historical weather observation data you want to retrieve.

Response Format

The response format for each endpoint will vary depending on the specific data being retrieved. However, most responses will be in JSON format and will include relevant weather information such as temperature, humidity, wind speed, and more.

Error Handling

If there are any errors with your API request, you will receive an appropriate HTTP status code along with an error message in the response body.

Rate Limits

To ensure fair usage of the Greenaware API, rate limits are enforced on API requests. Please refer to our API documentation for details on rate limits and how to handle rate limit errors.

Get Started

Ready to get started? Sign up for an API key and explore the full capabilities of the Greenaware API. If you have any questions or need assistance, don't hesitate to contact our support team.
